# ガイドライン

このガイドラインでは、言語共通、言語別などに分け、一般に遵守すべき規則を示す。
すべてのケースにおいて、このガイドラインに記述されている限りではない場合があるが、必ず以下の記述を考慮すること。

## 全言語共通

- 型や関数は再利用を考慮して、同じようなコードを繰り返し記述しないようすること。
- 一つのファイル、モジュールには、ファイル名、モジュール名に即した内容のみを記述し、それぞれのコード量が肥大しないよう、適切に分割すること。
- コメントの言語はプロジェクトの現状に即したものにすること。英語のコメントが多ければ英語、日本語が多ければ日本語といった具合である。
- 無駄にコメントを書く必要はないが、複雑で自明ではないコードへのコメントは記述すること。公開APIへのドキュメントコメントは、関数のシグネチャからわからない範囲のみ記述すればよい。
- テストにはこまめにコメントを記述することを求める。
- 自明なテストの記述は避けること。
- 非公開の関数に関するテストは、必要な場合のみにすること。

## 各言語について

各言語について従うべき規約は、スキルとして記述されている。
記述している言語毎にスキルがある場合は必ず参照し、規約に従ったコードを記述すること。
