## このスキルを使うべきとき

このスキルは、Pythonを記述する時に常に参照し、本文書に記述されたPythonの
ベストプラクティスおよびユーザのコーディングスタイルに従って開発を
進めるためのものである。

## instruction

- 基本的には一般的なPythonのベストプラクティスに従うこと。
- ワークスペースで利用しているパッケージマネージャを確認し、特にない場合はuvを用いて、スクリプトの実行には `uv run` を用いること。
- コードのフォーマットはblackを用いて、書き終えた際にフォーマットを実行すること。
- パッケージの追加は、 `pypi.org` などで最新のパッケージを調べて、依存関係が解決可能な範囲で最新のものを利用すること。

### 型注釈

- 基本的にすべての関数に型注釈をつけること。
- 定期的に `ty check` を用いて型検査すること。
- `import typing` はできるだけ避け、 `list` などの標準の型注釈を使うのが好ましい。
